## Reporting a Vulnerability

**Please do not report security vulnerabilities through public GitHub issues.**

Instead, please report them via email to: **security@agentsentinel.dev**

Include:
- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

### What to Expect

- **Acknowledgment:** Within 24 hours
- **Initial Assessment:** Within 72 hours
- **Resolution Timeline:** Depends on severity
  - Critical: 24-48 hours
  - High: 1 week
  - Medium: 2 weeks
  - Low: Next release

### Responsible Disclosure

We follow responsible disclosure practices:
1. Report is received and acknowledged
2. We investigate and confirm the issue
3. We develop and test a fix
4. We release the fix and publish an advisory
5. After 90 days, the reporter may publicly disclose

## Security Best Practices

When using AgentSentinel:

1. **Keep Updated:** Always use the latest version
2. **API Keys:** Never expose API keys in code
3. **Canary Tokens:** Use canary tokens in production
4. **Block Threshold:** Set appropriate threshold for your use case
5. **Monitor Alerts:** Act on behavioral anomaly alerts
